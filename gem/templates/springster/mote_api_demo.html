Mote API demo

Mote provides an HTTP API, making it possible to integrate Mote into nearly any system. Here we use the JavaScript API wrapper to render components directly into your browser.

Render the Springster Version 1 search form. This is the one used by the Springster Global site.

Render the Malawi Version 1 search component.

We're working on a Springster Version 2 search component. It's not available in the Springster Global site yet but we can plan our components ahead independent of the code base.

That button text needs to change. Luckily Mote is *data driven*. Let's feed the component some data!

How does this dark magic work?

// Get an API instance
var mote_api = new MoteAPI('/mote/api/');

// Push commands
mote_api.push(
    'springster.browser.atoms.search',
    {},
    '#one'
);
mote_api.push(
    'malawi.browser.atoms.search',
    {},
    '#two'
);
mote_api.push(
    'springster2.browser.atoms.search',
    {},
    '#three'
);
mote_api.push(
    'springster.browser.atoms.search',
    {"children": {"button": {"attrs": {"value": "Click me to search!"}}}},
    '#four'
);

// Run the commands
mote_api.run();
